git merge

将两个分支的内容进行合并，在被合并分支后创建一个合并后的新节点

git branch

查看本地仓库分支和远程仓库分支

git rebase

将一个分支上的提交逐次复制到另一个分支的末尾，从而形成一条线性的提交历史

上述命令会将 feature 分支上的所有提交复制到 main 分支的最新提交之后，然后将 feature 分支的指针移动到这些新提交的末尾。

git fetch

git fetch 命令的主要作用是从远程仓库下载新的提交、分支和标签等信息到本地仓库，但不会自动合并到当前的工作分支。

  • 与 git pull 的对比

    • git pull 实际上是 git fetch 和 git merge 的组合操作。它先执行 git fetch 从远程仓库获取最新信息，然后自动将这些信息合并到当前工作分支。
