一、环境准备

  1. 安装 Node.js
    • 需不低于 10.13 版本，建议使用 12.0 以上。
    • 官网下载地址：https://nodejs.org/zh-cn。
    • 验证安装：命令行输入 node -v 和 npm -v，显示版本号即成功。
  2. 安装 Git
    • 用于代码版本管理和部署。
    • 官网下载地址：https://git-scm.com/。
    • 验证安装：命令行输入 git --version。

二、GitHub 仓库配置

  1. 创建仓库
    • 仓库名必须为 用户名.github.io（例如 john.github.io）。
    • 勾选 Initialize with a README（可选）。
  2. 配置 SSH 密钥
    • 生成密钥：ssh-keygen -t rsa -C "GitHub 注册邮箱"，一路回车。
    • 添加密钥到 GitHub：
      • 复制 ~/.ssh/id_rsa.pub 内容。
      • 进入 GitHub Settings → SSH and GPG keys → New SSH key。

七、自动化部署

使用 GitHub Actions 实现 Hexo 博客自动化构建和部署

（一）准备工作

  1. 代码仓库拆分（可选但推荐）
    • 源码仓库：存放 Hexo 项目源码（如 hexo-source 分支或单独仓库）。
    • 部署仓库：存放生成的静态文件（即 用户名.github.io 仓库）。
    • 如果未拆分，可直接将源码和生成文件放在同一仓库的不同分支（如 main 分支存源码，gh-pages 分支存静态文件）。
  2. 生成 Personal Access Token（若需跨仓库部署）
    • 进入 GitHub → Settings → Developer settings → Personal access tokens → Generate new token。
    • 勾选 repo 和 workflow 权限，生成后复制 Token 值备用（需保存，后续不可见）。

（二）配置 GitHub Actions

  1. 在源码仓库中创建 Workflow 文件

（如果已经为你的仓库 用户名/用户名.github.io 配置了SSH密钥，这里生成的步骤可跳过）

  1. 生成密钥对

    • 按提示选择保存路径（默认 ~/.ssh/id_ed25519）。
    • 设置密码（可选，但建议留空以便自动化流程无交互）。

  2. 获取密钥内容

    • 私钥：打开生成的 id_ed25519 文件，复制全部内容。
    • 公钥：打开生成的 id_ed25519.pub 文件，复制全部内容。

（四）在 GitHub 中配置密钥

这里博主选择分开部署仓库 + 源码仓库的方式

  1. 部署仓库添加公钥
    • 进入部署仓库（如 用户名.github.io）的 Settings → Deploy Keys。
    • 点击 Add deploy key：
      • Title：自定义名称（如 HEXO_DEPLOY）。
      • Key：粘贴公钥内容（id_ed25519.pub）。
      • 勾选 Allow write access：必须开启，否则无法推送文件。
  2. 源码仓库配置私钥
    • 进入源码仓库的 Settings → Secrets and variables → Actions。
    • 点击 New repository secret：
      • Name：DEPLOY_KEY（与 Actions 配置中的变量名一致）。
      • Value：粘贴私钥内容（id_ed25519）。

（五）验证流程

  1. 推送代码到源码仓库，观察 Actions 是否自动运行。
  2. 若部署成功，目标仓库（如 用户名.github.io）的对应分支会更新 public 目录下的文件。
  3. 访问 https://用户名.github.io 确认页面更新。

常见问题

  • 部署失败：检查 _config.yml 格式（冒号后需空格）。
  • 图片路径错误：安装插件 hexo-asset-image。

通过以上步骤，您可以在 GitHub 上免费托管一个高性能的静态博客。如需进一步优化，可参考 Hexo 官方文档。
